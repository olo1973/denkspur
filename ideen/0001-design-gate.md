# Idee 0001: Design-Gate für GUI-Vorhaben

Befördert zu: [Entscheidung 0003](../entscheidungen/0003-design-gate-abnahme-am-guenstigsten-medium.md)

Beobachtung aus der Praxis: Bei Dev-Projekten ergibt es Sinn, die
Benutzeroberfläche zunächst in Figma zu entwerfen und **visuell abzunehmen**,
bevor Code generiert wird. Ein abgelehntes Layout kostet am Bild Minuten, am
fertigen Code Stunden — und hinterlässt dort Altlasten.

Offen: Ist das ein Spezialfall (Figma für GUIs) oder steckt ein allgemeines
Prinzip dahinter, das in den Vertrag der Methode gehört?
